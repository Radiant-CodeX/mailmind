### For Engineers / Technical Audience

**Architecture (Slide 9):**

1. **Deterministic Fallbacks**
   - LLM failures → fallback to 5 rule-based scoring tools
   - Queue unavailable → fallback to in-memory queue
   - DB unavailable → graceful no-op (metrics still recorded to in-memory fallback)
   - **Result:** Zero hard failures; system degrades gracefully

2. **PII-First Security**
   - Mask before LLM invocation (regex + Presidio)
   - Restore after LLM returns (token mapping)
   - Neutralize hallucinated tokens (LLM invents `[PERSON_2]` not in mapping → replace with "there")
   - **Result:** No raw PII in logs; compliant with GDPR/CCPA

3. **Human-in-the-Loop**
   - CRITICAL (score ≥75) → **GATE** mode (requires `POST /api/commitments/confirm`)
   - HIGH/MEDIUM/LOW → **SUGGEST** mode (suggestions only)
   - **Result:** AI speeds up routine decisions; humans keep control of high-stakes ones

4. **Tone DNA (RAG)**
   - Index user's 50+ sent emails in ChromaDB
   - On triage, retrieve 3 most similar past emails
   - Inject as few-shot examples into draft prompt
   - **Result:** Drafts sound like the user (not generic OpenAI)

5. **Observable Pipeline**
   - Prometheus metrics on each node (latency, success rate, LLM fallback %)
   - Audit log on all emails (who, what, when, why)
   - SLA tracking (% meeting <1.5s triage, <10s enrichment)
   - **Result:** Data-driven optimization; compliance audit trail

6. **Split SLAs**
   - **Triage** (ingest → triage → gate) = synchronous, <1.5s SLA (user-facing)
   - **Enrichment** (commitment → calendar → RAG) = asynchronous, <10s SLA (deferred worker)
   - **Result:** Fast UI response; rich analysis in background

**Code Quality:**
- Fully typed (TypeScript + Python with type hints)
- 42 unit tests (all passing)
- CI/CD pipeline (lint, type-check, tests, Docker build on main)
- No secret leaks (Azure Key Vault, .env in .gitignore)

"The system just scored 5 dimensions:

- **Thread Risk:** 88% — If we ignore this, we lose a major client contract

These 5 axes aren't hardcoded. GPT-4o calculates them fresh for *this specific email* because different emails weight different axes differently. A newsletter might score high on deadline if it's time-sensitive, but low on authority and action. This email scores high on all of them.

The composite score is 78 out of 100 → **CRITICAL priority**. The system is saying: *'Human, you need to look at this one.'*"

2. 'Arrange signature ceremony' — Confidence 88% — Deadline: tomorrow
3. 'Coordinate with legal' — Confidence 92% — Deadline: tomorrow 10AM

So the system isn't just extracting tasks blindly. It's checking the calendar and saying: *'By the way, this task has a scheduling problem. You might want to adjust the deadline or delegate the call.'*"

*[Switch to Slide 6: AI-Generated Draft]*

## Troubleshooting

### Pipeline doesn't show up
- ✅ Make sure `METRICS_ENABLED=true` in `.env`
- ✅ Make sure an email was selected (not loading)
- ✅ Check browser console for errors

### Metrics endpoint returns 404
- ✅ Backend must be running: `python -m uvicorn app.main:app --reload`
- ✅ Metrics must be enabled in settings

### Presentation mode slides are blank
- ✅ Make sure all props are passed (email, triageResult, commitments, etc.)
- ✅ Check that data is loaded (not in loading state)

### Presentation mode doesn't open
- ✅ Make sure you're clicking on the presentation button
- ✅ Or press **P** while email detail is open
- ✅ Check for console errors

## What to Say When...

### When someone asks "What if the AI is wrong?"
**Answer:** "Great question. Two things:
1. We have deterministic fallbacks. If the AI is unavailable, we use rule-based scoring. If the AI gives a low-confidence answer, we flag it for human review.
2. For CRITICAL emails, humans approve before actions proceed. The AI is a suggestion engine, not an autonomous agent."

### When someone asks "What about privacy?"
**Answer:** "PII-first design. We mask emails before sending them to the LLM. Phone numbers, emails, names → replaced with placeholders. After the LLM analyzes, we restore the real data. The LLM never sees raw PII."

### When someone asks "How does it learn my tone?"
**Answer:** "Tone DNA uses RAG (Retrieval-Augmented Generation). We index your sent emails as vectors in a database. When you need a reply, we find similar past emails you sent and inject them as examples into the draft prompt. The LLM learns your style from those examples."

### When someone asks "How fast is it?"
**Answer:** "1.4 seconds end-to-end for the full pipeline. Triage (the user-facing part) is <1.5 seconds with a 99% SLA. If you're reading this email, the triage is already done."

### When someone asks "What if I don't want to approve?"
**Answer:** "CRITICAL emails require your approval. HIGH/MEDIUM/LOW are suggestions only. You can always reject a suggestion or edit a draft. The system proposes; you decide."
